## 二、商业背景与研究问题

### 2.1 商业背景

信贷业务是商业银行和消费金融公司的核心利润来源。准确识别违约风险是控制资产质量的关键。传统信用评估（专家判断、评分卡）依赖有限变量和人工经验；机器学习能从多维数据中自动学习复杂违约模式，实现更精准的风险识别。

现代金融机构不仅是"风险规避者"，更是"价值创造者"。在控制风险的前提下识别高价值客户、实施差异化策略，是金融科技时代数据驱动决策的核心命题。

### 2.2 三个研究问题

| 层次 | 问题 | 方法 | 业务价值 |
|:---|:---|:---|:---|
| **识别** | 哪些因素最能预测客户违约？ | 随机森林特征重要性分析 | 识别核心风险驱动因子 |
| **预测** | 能否构建高精度违约预测模型？ | 逻辑回归 + 随机森林对比 | 为信贷审批提供自动化工具 |
| **分层** | 如何平衡"风险"与"价值"？ | 风险-价值四象限矩阵 | 差异化客户经营策略 |

## 三、数据说明

### 3.1 数据来源

- **数据集**：天池金融风控贷款违约预测
- **平台**：阿里云天池 × Datawhale
- **描述**：来自某信贷平台的真实贷款记录，总数据量超过120万条
- **下载**：https://tianchi.aliyun.com/dataset/173753

### 3.2 数据规模

| 指标 | 数值 |
|:---|:---|
| 抽样样本量 | 30,000 条（从80万训练集中分层抽样） |
| 变量数 | 47 列（含15列匿名特征 n0~n14） |
| 数据集划分 | 训练集80万 / 测试集A 20万 / 测试集B 20万 |
| 违约比例 | 约 20%（类别不平衡） |

### 3.3 关键字段速查

| 字段 | 含义 | 类型 |
|:---|:---|:---|
| `loanAmnt` | 贷款金额 | 连续 |
| `term` | 贷款期限（年） | 数值 |
| `interestRate` | 贷款利率 | 连续 |
| `installment` | 分期付款金额 | 连续 |
| `grade` / `subGrade` | 贷款等级（A-G） | 分类 |
| `employmentLength` | 就业年限 | 分类（含缺失） |
| `homeOwnership` | 房屋所有权（0=租用,1=自有,2=抵押） | 分类 |
| `annualIncome` | 年收入 | 连续 |
| `dti` | 债务收入比 | 连续 |
| `delinquency_2years` | 近2年逾期30天以上次数 | 数值 |
| `ficoRangeLow/High` | FICO信用分范围 | 连续 |
| `revolBal` / `revolUtil` | 信贷周转余额/循环额度利用率 | 连续 |
| `n0` ~ `n14` | 匿名特征（脱敏处理） | 连续（含缺失） |
| `isDefault` | **目标变量** | 0=未违约，1=违约 |

### 3.4 数据质量注意事项

- **缺失值**：`employmentLength`（填充"未知"）、`n0~n14`（中位数填充）、`dti`/`pubRecBankruptcies`/`revolUtil`（少量缺失，中位数填充）
- **类别不平衡**：违约约19.6%，建模时使用 `class_weight='balanced'` 处理
- **极端值**：dti 存在1条999%占位符错误→中位数替换；annualIncome 最高180万→99分位截断至25万
- **匿名特征**：n0~n14 为脱敏后的行为计数特征，无业务含义，但可用于提升模型精度
- **编码方案**：grade/subGrade/regionCode/title → LabelEncoder；purpose（14类无序）→ OneHot

### 4.4 风险-价值矩阵构建方法（详细）

这是项目最重要的差异化模块，具体计算步骤如下：

**步骤一：计算风险评分**
```python
# 使用随机森林预测违约概率
risk_score = rf_model.predict_proba(X_test)[:, 1]  # 取正类（违约）概率
```

**步骤二：计算价值评分**
```python
# 综合额度水平和消费活跃度
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()

# 收入因子：年收入越高，客户潜在价值越大
income_score = scaler.fit_transform(df[['annualIncome']])

# 额度因子：贷款金额越高，平台收益越大
loan_score = scaler.fit_transform(df[['loanAmnt']])

# 价值评分 = 收入因子 × 0.5 + 贷款金额因子 × 0.5
value_score = 0.5 * income_score.flatten() + 0.5 * loan_score.flatten()
```

**步骤三：四象限分类**
```python
# 中位数二分
risk_median = np.median(risk_score)
value_median = np.median(value_score)

# 分类逻辑
# 低风险 + 高价值 = 明星客户（重点维护）
# 低风险 + 低价值 = 潜力客户（提升活跃度）
# 高风险 + 高价值 = 挽留客户（谨慎经营）
# 高风险 + 低价值 = 拒绝客户（控制风险）
```
