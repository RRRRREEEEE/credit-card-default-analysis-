## 三、上下文管理机制

### 3.1 问题：对话太长我会"忘记"

本项目预计产生大量代码、数据和反复讨论。当对话超过一定长度，我会丢失早期的上下文。

### 3.2 解决方案：文件化一切关键信息

以下文件构成我的"外部大脑"，我每次对话开始时**必须读取**：

| 文件 | 内容 | 更新频率 | 读取优先级 |
|:---|:---|:---|:---:|
| `RULES.md` | 你给我的所有指令、偏好、约束 | 你随时改 | **最高** |
| `项目计划书.md` | 项目总体计划 | 几乎不变 | 高 |
| `CHANGELOG.md` | 每日做了什么、做到哪了 | 每次会话结束 | 高 |
| `DECISIONS.md` | 关键决策和变更原因 | 有重大决策时 | 中 |

### 3.3 会话启动流程

每次新对话开始，我会：
1. 读取 `RULES.md` — 知道你的最新指令
2. 读取 `CHANGELOG.md` — 知道项目当前状态
3. 读取 `DECISIONS.md` — 知道之前为什么做了某些选择
4. 快速扫 `项目计划书.md` — 确认整体方向
5. 向你汇报："我现在知道项目在 X 阶段，上次做到 Y，今天继续 Z"

### 3.4 代码上下文

所有代码存在 `.ipynb` 文件中。我通过读取 notebook 来获取当前代码状态，不需要你把代码贴给我。

## 四、指令与执行机制

### 4.1 RULES.md — 甲方指令簿

你随时可以用自然语言对我说"记住，以后 X"，我会写入 `RULES.md`。例如：
- "记住，数据列名统一用小写加下划线"
- "记住，论文正文不要超过 5000 字"
- "记住，每次改代码前先备份"

我会在每次行动前检查 `RULES.md`，确保不违反你的任何规则。

### 4.2 执行前确认原则

| 操作类型 | 是否需你确认 | 说明 |
|:---|:---:|:---|
| 读取文件、搜索 | 不需要 | 只读操作 |
| 写入新文件 | 不需要 | 新文件不破坏现有内容 |
| **修改已有代码** | **需要确认** | 先说明改什么、为什么，你同意再动手 |
| **修改项目计划** | **需要确认** | 变更方向必须你点头 |
| **删除文件** | **需要确认** | 走回收站 |
| **论文正文撰写** | **需要确认** | 先给大纲/草稿，你审阅后再定稿 |

### 4.3 你的控制指令

| 指令 | 效果 |
|:---|:---|
| "停" / "等等" | 我立即停止当前操作，等你指示 |
| "不改了，保持原样" | 撤销我刚刚的建议，回到之前状态 |
| "这个方案不行，换个思路" | 我放弃当前方向，提新方案 |
| "记住，以后 X" | 将我写入 RULES.md |
| "更新日志" | 我将本次会话的所有变更写入 CHANGELOG.md |
| "备份当前状态" | 我将关键文件复制到 backups/ 目录 |

## 五、版本控制与回滚机制

### 5.1 Git 仓库（推荐启用）

```bash
cd 高级商业数据分析
git init
git add -A
git commit -m "初始状态：项目计划完成"
```

**每次里程碑完成后提交一次**，标签清晰：

| 提交时机 | Commit Message | Tag |
|:---|:---|:---|
| 初始状态 | `init: 项目计划完成` | — |
| 阶段一完成 | `M1: 数据预处理完成` | `m1-data` |
| 阶段二完成 | `M2: 建模分析完成` | `m2-model` |
| 阶段三完成 | `M3: Tableau看板完成` | `m3-tableau` |
| 阶段四完成 | `M4: 论文初稿完成` | `m4-paper` |
| 最终提交 | `M5: 全部材料打包` | `m5-final` |

**回滚**：`git checkout <tag> -- <file>` 即可恢复任意文件到任意里程碑。

### 5.2 手动备份（备选）

如果不想用 Git，每次大改前手动复制到 `backups/` 目录：
```
backups/
├── 20260704_初始状态/
├── 20260705_M1完成/
├── 20260707_M2完成/
└── ...
```

### 5.3 Notebook 特别保护

Jupyter Notebook 的输出会随运行改变文件。规则：
- 每次对 notebook 做大改前，先复制一份到 backups
- 修改后先确认代码逻辑正确，再运行
- 绝对不删除旧版本的 notebook cell，只新增或修改
